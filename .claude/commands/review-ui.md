# UI/UX 코드 리뷰

선택된 코드 또는 파일에 대해 UI/UX 관점에서 상세한 코드 리뷰를 수행합니다.

## 리뷰 대상

$ARGUMENTS

## 리뷰 항목

다음 관점에서 코드를 분석하고 개선점을 제시해주세요:

### 1. 접근성 (Accessibility)
- ARIA 속성 적절성
- 키보드 네비게이션 지원
- 스크린 리더 호환성
- 색상 대비 및 가독성
- focus 상태 표시

### 2. 반응형 디자인 (Responsive Design)
- 모바일/태블릿/데스크톱 대응
- Tailwind 반응형 클래스 활용
- 터치 타겟 크기 (최소 44x44px)
- 유동적 레이아웃

### 3. 사용자 경험 (UX)
- 로딩 상태 처리
- 에러 상태 표시
- 빈 상태 (Empty State) 처리
- 피드백 제공 (hover, active, disabled)
- 일관된 인터랙션 패턴

### 4. 시각적 일관성 (Visual Consistency)
- 디자인 토큰 활용 (CSS 변수)
- shadcn/ui 컴포넌트 활용도
- 간격 및 정렬 일관성
- 타이포그래피 계층 구조

### 5. 성능 (Performance)
- 불필요한 리렌더링
- 이미지 최적화 (next/image)
- 레이아웃 시프트 방지
- 애니메이션 성능

## 출력 형식

각 항목에 대해 다음 형식으로 피드백을 제공해주세요:

- ✅ **잘된 점**: 유지해야 할 좋은 패턴
- ⚠️ **개선 권장**: 개선하면 좋을 부분
- ❌ **수정 필요**: 반드시 수정해야 할 문제

마지막에 전체 점수(10점 만점)와 우선순위별 개선 사항 요약을 제시해주세요.
